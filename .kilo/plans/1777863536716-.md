# Docsify Artifact Visualizer — Dependency-Ordered Implementation Tasks

## Phase 0: Project Setup (No Dependencies)
**Milestone 1 — Foundation**

- [ ] Create Vite project with vanilla JS target
- [ ] Add `vite-plugin-singlefile` to inline JS/CSS into one HTML file
- [ ] Add custom `injectSource` or equivalent build-time source capture for self-replicating export
- [ ] Set up `index.html` as the single deployable app shell
- [ ] Establish `src/` module layout (`shell.js`, `import.js`, `config-ui.js`, `plugin-registry.js`, `presets.js`, `preview.js`, `export.js`, `themes.js`, `renderers/`)
- [ ] Establish `styles/` layout (`config-ui.css`, `renderers.css`, `tokens.css`)
- [ ] Create `src/demo.md` with built-in demo content showcasing all renderers

## Phase 1: App Shell & State (Depends on Phase 0)
**Milestone 1 — Foundation**

- [ ] Implement app boot sequence (`src/shell.js`)
- [ ] Implement `localStorage` state manager (`avz_content`, `avz_config`, `avz_mode`, `avz_meta`, `avz_seed_id`, `avz_custom_plugins`)
- [ ] Add demo markdown content for first-run experience (load when no saved session)
- [ ] Implement config/preview mode switch with `?mode` URL parameter support
- [ ] Add session restore on load (check localStorage, restore mode/content/config)
- [ ] Build basic config screen layout (Import + Configure panels)

---

## Phase 2: Import UX (Depends on Phase 1)
**Milestone 2 — Import and Preview**

- [ ] Implement global clipboard paste handling (`src/import.js`) — paste event, check for files first, fallback to text
- [ ] Implement file browse import — `<input type="file" accept=".md,.txt,.markdown">`
- [ ] Implement drag-and-drop import — drop zone with `dragover` highlight and `drop` handling
- [ ] Implement URL fetch import — fetch URL, handle CORS failure with inline warning
- [ ] Implement `handleMarkdownContent(string)` — update state, persist to localStorage, show preview
- [ ] Add import metadata tracking (source, filename, timestamp, byte length)
- [ ] Add size check with warning threshold (~2MB)

---

## Phase 3: Config UI (Depends on Phase 1)
**Milestone 3 — Presets and Plugins**

- [ ] Build preset picker UI with 6 presets (spec-doc, api-doc, architecture, data-report, dev-journal, minimal)
- [ ] Implement preset selection with overwrite behavior (no merge logic)
- [ ] Create plugin registry (`src/plugin-registry.js`) with 9 built-in plugins (kroki, shiki, copy-code, flex-alerts, tabs, termynal, drawio, pagination, zoom-image)
- [ ] Build plugin manager UI — toggle plugins, show verified/unverified badges
- [ ] Add per-plugin config panels (e.g., Kroki server URL with test connection button)
- [ ] Implement "Add plugin URL" flow for custom plugins
- [ ] Add custom plugin verification via sandbox iframe and postMessage
- [ ] Build token editor with 8 core controls (color pickers, text inputs, range sliders)
- [ ] Add "Advanced" expander for remaining ~32 docsify-themeable variables
- [ ] Add unsaved-change indicator for advanced tokens
- [ ] Add Reimport confirmation dialog

## Phase 8: Export Module (Depends on Phase 1, 4, 5, 7)

- [ ] Implement seed script generation with inline markdown + config JSON
